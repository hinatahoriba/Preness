# 機能仕様

## 1. 認証機能（Devise）

### 基本機能

- 新規登録（ユーザー名、メールアドレス、パスワード）
- ログイン / ログアウト
- パスワードリセット
- 利用規約・プライバシーポリシーへの同意（必須チェックボックス）

### メール認証

- メール認証は必須（`confirmable` モジュール使用）
- 未認証ユーザーはログイン不可
- 認証メールの再送信機能あり

### ユーザー名

- 必須項目
- 重複可能（UNIQUE制約なし）
- 変更可能

### メールアドレス

- 変更可能
- 変更時は再度メール認証が必要（Deviseデフォルト動作）

### セキュリティ設定

| 項目 | 設定 |
|---|---|
| セッションタイムアウト | 24時間 |
| ログイン試行回数制限 | なし |
| 二段階認証 | なし |
| ソーシャルログイン | なし |

### アカウント管理

- アカウント削除機能: なし
- プロフィール情報: 不要

## 2. サブスクリプション機能

### 概要

- セクション別演習へのアクセスをサブスクリプションで管理
- 無料トライアル（7日間）→ 自動的にプレミアムプラン（月額980円）に移行
- 決済: Stripe Checkout
- 管理: Stripe Customer Portal

### Stripe Customer 管理方針

- `users.stripe_customer_id` でStripe CustomerをDB管理
- サブスク開始時または模擬試験購入時に初回作成（`stripe_customer_id` がNULLの場合のみ）
- 既に `stripe_customer_id` が存在する場合は新規作成せず共用
- サブスクと模擬試験購入で同じCustomerを使い回す

### トライアル開始フロー

**パターンA: LPから登録**
```
LP → トライアル開始ボタン → 新規登録 → メール認証 → ログイン
→ Stripe Checkout（カード登録） → Webhook受信 → トライアル開始
```

**パターンB: 直接登録**
```
新規登録 → メール認証 → ログイン → マイページ → トライアル開始ボタン
→ Stripe Checkout（カード登録） → Webhook受信 → トライアル開始
```

**トライアル開始前の制限**
- セクション別演習には一切アクセス不可
- マイページでトライアル開始を促すCTA表示

### 無料トライアル

| 項目 | 仕様 |
|---|---|
| 期間 | 7日間 |
| 利用制限 | 1日3セットまで（user_exercisesで管理） |
| カード情報 | トライアル開始時に必須 |
| 終了後 | 自動的にプレミアムプラン（月額980円）に移行 |
| 利用回数 | 1ユーザーにつき1回のみ |

**トライアル済みユーザーへの対応**
- トライアル開始ボタン押下時に `subscriptions.status` を確認
- `none` 以外の場合は「既にトライアルをご利用済みです」エラー表示

**1日3セットの制限詳細**
- カウント単位: question_set（大問セット）
- カウントタイミング: セットを開いた時点（`UserExercise.create` 実行）
- リセット: 日本時間0:00（`consumed_on` の日付条件で自然にリセット）
- 同一 `question_set_id` が既存なら追加消費なし（途中再開対応）
- 制限到達時: プレミアムプランへのアップグレード誘導を表示

### プレミアムプラン

| 項目 | 仕様 |
|---|---|
| 月額料金 | 980円（税込） |
| 提供内容 | セクション別演習（無制限） |
| 更新 | 自動更新（毎月） |
| 初回課金 | トライアル終了時に自動課金（Stripeデフォルト） |
| 解約 | いつでも可能（次回更新日まで利用可能） |

**解約後の扱い**
- 次回更新日まで: プレミアム機能利用可能
- 更新日以降: セクション別演習に完全アクセス不可

### 支払い失敗時の処理

| 項目 | 仕様 |
|---|---|
| 自動リトライ | Stripeデフォルト設定 |
| 猶予期間 | 3日間 |
| 猶予期間終了後 | セクション別演習に完全アクセス不可 |
| ユーザー通知 | Resendでメール通知 |

**再加入フロー**
- 新規サブスクリプションとして処理
- トライアルは利用不可（既利用のため）
- 直接プレミアムプラン（月額980円）に加入

### サブスクリプション状態遷移

| 状態 | 説明 | セクション別演習 |
|---|---|---|
| none | 未加入（トライアル未開始） | アクセス不可 |
| trial | トライアル期間中 | 1日3セットまで |
| premium | プレミアム会員 | 無制限 |
| cancelled | 解約済み（更新日まで利用可能） | 更新日まで無制限 |
| expired | 期限切れ | アクセス不可 |

### Stripe Webhook

**署名検証:** `Stripe-Signature` ヘッダーを `STRIPE_WEBHOOK_SECRET` で検証。検証失敗時は `400 Bad Request` を返す。

**処理失敗（DBエラー等）:** `500` を返すことでStripeが自動再試行する。冪等性を保つ実装が必要。

| イベント | 処理内容 |
|---|---|
| `checkout.session.completed` | トライアル開始または模擬試験購入の完了処理 |
| `customer.subscription.updated` | サブスクリプションステータスの変更反映 |
| `customer.subscription.deleted` | 解約・期限切れの反映 |
| `invoice.payment_failed` | 支払い失敗の検知・Resendでメール通知 |

**冪等性の実装指針**
- `stripe_subscription_id` や `stripe_checkout_session_id` を使って既存レコードの存在確認
- `upsert` または `find_or_initialize_by` で重複処理を防ぐ

## 3. 模擬試験購入機能

### 概要

| 項目 | 仕様 |
|---|---|
| 購入形態 | 単発購入 |
| 価格 | 1,500円（税込・固定） |
| 決済方法 | Stripe Checkout |
| 受験回数 | 1回きり（再受験不可） |

### 購入フロー

1. マイページの購入ボタン押下
2. `created_at` が最も古い未購入の模擬試験を自動選択（ユーザーが選ぶUIは不要）
3. Stripe Checkoutにリダイレクト
4. `checkout.session.completed` Webhook受信 → `user_exams` レコードを作成し、ステータスを `paid` と設定する（※離脱時のゴミデータを防ぐため、Checkout遷移前にはレコードを作成しない）

### 購入履歴（マイページ表示内容）

- 購入済み模擬試験タイトル
- 受験状態（未受験 / 受験中 / 受験済み）
- スコア（受験済みのみ）

### 模擬試験とサブスクリプションの関係

- 模擬試験購入とサブスクリプションは完全に独立
- 模擬試験のみ購入したユーザーはセクション別演習にアクセス不可
- トライアル未利用であればトライアル開始は可能

## 4. 出題セクション構成・問題数

### Section 1: Listening Comprehension（リスニング）

| パート | 内容 | 問題数（模擬試験） |
|---|---|---|
| Part A: Short Conversations | 短い会話を聞き質問に答える | 30問（1セット1問×30） |
| Part B: Longer Conversations | 長めの会話、複数問 | 8問（2会話×4問） |
| Part C: Talks | 講義・スピーチ、複数問 | 12問（3トーク×4問） |

**合計: 50問**

### Section 2: Structure and Written Expression（文法）

| パート | 内容 | 問題数（模擬試験） |
|---|---|---|
| Part A: Structure | 空所補充 | 15問 |
| Part B: Written Expression | 正誤判定（下線4箇所から1選択） | 25問 |

**合計: 40問**

### Section 3: Reading Comprehension（読解）

| パート | 内容 | 問題数（模擬試験） |
|---|---|---|
| Passages | パッセージを読み複数問に答える | 50問（5〜6パッセージ×8〜10問） |

**合計: 50問**

> 上記問題数は模擬試験の標準構成。セクション別演習はパート単位で独立して追加可能なため、問題数は可変。

## 5. セクション別演習

### 問題一覧ページ（`/practice_exams`）

- セクション見出し（リスニング → 文法 → 読解の順）配下に、question_set単位でセット一覧
- **セット名自動生成ルール:**
  ```
  {セクション名} {パート名} - セット{display_order}
  例: リスニング Part A - セット1
      文法 Part B - セット3
      読解 - セット2
  ```
- サブスク状態に応じたロック表示:
  - `none`: 全セットにロックアイコン + トライアル開始誘導
  - `trial`: 1日3セット制限の残数表示（例: 「本日あと2セット」）
  - `premium` / `cancelled`（期間内）: 全セット利用可能
  - `expired`: ロック表示

### 解答フロー

- 全問解いてから一括送信 → 自動採点
- 同じセクション内であれば前の問題に戻って回答変更可能
- **リロード対応:** 回答をブラウザの `sessionStorage` に保存（サーバー側には保存しない）
- リロード時: `sessionStorage` から回答を復元
- 解き直し: 何度でも可能（スコアの履歴は保存しない）
- 制限時間なし

### 音声再生ルール

- 再生回数: 1回のみ
- 操作: 再生ボタンのみ（一時停止・シークバーなし）
- 再生後: ボタン無効化 + 「再生済み」テキスト表示

### 採点結果画面

- 正答率（正解数 / 全問数、パーセンテージ、プログレスバー）
- 間違えた問題のみをカード表示（正解した問題は非表示）
- 各カード: 問題文、あなたの回答（赤）、正解（緑）、解説

## 6. 模擬試験

### 受験フロー

- セクション順は固定: **Listening → Structure → Reading**
- セクション間の移動: 「次のセクションへ」ボタンで遷移（確認ダイアログあり）
- 前のセクションに戻ることは不可
- 同じセクション内であれば前の問題に戻って回答変更可能

### 制限時間

| セクション | 制限時間 |
|---|---|
| Listening Comprehension | 約30分（1,800秒） |
| Structure and Written Expression | 25分（1,500秒） |
| Reading Comprehension | 55分（3,300秒） |

- タイマー管理: フロントエンド（JavaScript）のみ
- 残り5分で文字色を赤に変更、残り1分でパルスアニメーション
- 時間切れ時: 自動で現在の回答を送信

### 途中離脱・再開

- `in_progress` 状態で無期限に再開可能
- **再開時は常にSection 1（Listening）から表示**
- 既存の回答（answersテーブル）はそのまま復元される
- タイマーはフルリセット（ペナルティなし）
- 回答はセクション進行時にサーバーへ保存（answersテーブルに逐次保存）

### スコアレポート

- スコア計算: 単純な正答率ベース
- 表示内容: セクション別スコア・正答率、間違えた問題と解説（セクション別タブ切り替え）

## 7. 問題データの管理

### データ投入フロー

本アプリの問題データは、外部の専用アプリケーション（FastAPI）によって生成され、API経由で投入されます。

1. **問題生成:** FastAPI アプリで問題文、選択肢、音声を生成。
2. **音声アップロード:** FastAPI アプリが音声を S3 バケットにアップロードし、公開URLを取得。
3. **データ送信:** FastAPI アプリが本アプリの API エンドポイント（`/api/v1/mocks` または `/api/v1/exercises`）へデータを POST。
4. **データ保存:** 本アプリ側で受け取った受データを DB に保存（音声は `audio_url` として保存）。

### 管理方針

- 本アプリ側に問題編集用の管理画面は作成せず、投入されたデータを正として扱う。
- 問題の修正が必要な場合は、FastAPI 側で再生成・再投入（または DB の直接修正）を行う運用とする。
- 音声ファイルの実体は S3 にあり、本アプリは Active Storage を介さず直接 S3 の URL を参照して再生を行う。
