# 非機能要件

## 技術スタック

| 分類 | 技術 |
|---|---|
| バックエンド | Ruby on Rails 8.0.x |
| DB | PostgreSQL |
| 決済 | Stripe（Checkout / Customer Portal / Webhook） |
| 認証 | Devise |
| ファイル管理 | AWS S3（音声ファイル・直接参照） |
| フロントエンド | Tailwind CSS |
| 開発環境 | Docker |
| メール送信 | Resend（本番環境のみ） |

---

## 国際化・ローカライゼーション

- アプリの日本語化（i18n）
- 日時処理は全て日本時間（JST, UTC+9）
- タイムゾーン設定: `config.time_zone = "Tokyo"`

---

## アーキテクチャ方針

- サービス層を作成してコントローラーの肥大化を防ぐ（`app/services/` 配下）

---

## セキュリティ

| 項目 | 設定 |
|---|---|
| セッションタイムアウト | 24時間 |
| ログイン試行回数制限 | なし |
| 二段階認証 | なし |
| ソーシャルログイン | なし |
| APIキー認証 | 固定APIキー（環境変数 `CONTENT_API_KEY`） |
| Webhook署名検証 | `Stripe-Signature` ヘッダーを `STRIPE_WEBHOOK_SECRET` で検証 |

---

## ファイル管理

- 音声ファイルの実体は AWS S3 に保存。
- 本アプリは Active Storage を使用せず、DB に保存された `audio_url` を直接参照して再生を行う。
- パブリックアクセス（公開URL）または署名付きURLの運用とする。
- FastAPI 側で S3 へのアップロードを行い、本アプリへその URL を POST するフロー。

## 信頼性・冪等性

- Stripe Webhookの処理失敗時は `500` を返し、Stripeの自動再試行を利用
- 同一Webhookイベントが複数回届いても安全に処理できるよう冪等性を保つ実装が必要
  - `stripe_subscription_id` や `stripe_checkout_session_id` を使って既存レコードの存在確認
  - `upsert` または `find_or_initialize_by` で重複処理を防ぐ

---

## フロントエンドの注意事項

- タイマー管理はフロントエンド（JavaScript）のみで行う
- セクション別演習の回答はブラウザの `sessionStorage` に保存（サーバー側には保存しない）
- ターゲットデバイスはPCメイン（レスポンシブ対応はするが、PC体験を優先）
